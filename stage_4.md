# 📋 ЭТАП 4: Интеграция AI / ML
🎯 **Цель:** Безопасно подключить AI-модель, научиться формировать промпты, обрабатывать ответы и кэшировать результаты.

## 🛠️ Настройка
- [ ] Выбран вариант: OpenAI API (облако) или `ollama` (локально на Windows)
- [ ] Установлена библиотека-клиент (`openai` или `requests`/`httpx`)
- [ ] Ключи/эндпоинты вынесены в `.env`, НЕ в коде
- [ ] Настроен базовый обработчик ошибок API и `timeout`

## 📚 Теория и Практика
- [ ] Понятна структура промпта: роль, контекст, задача, формат вывода
- [ ] Освоен запрос к модели, парсинг JSON-ответа (`json.loads`)
- [ ] Изучено кэширование: `diskcache` или БД-таблица для ответов
- [ ] Понятны ограничения: rate limits, токенизация, галлюцинации
- [ ] Освоена валидация AI-ответов перед сохранением/выводом

## 📦 Проект: AI-Ассистент для учёбы
- [ ] Пользователь выбирает тему/период → формируется контекст из БД
- [ ] Промпт отправляется в модель, ответ парсится
- [ ] Ответ сохраняется в БД (история чата/рекомендаций)
- [ ] UI показывает ответ с индикацией загрузки/ошибки
- [ ] Реализован fallback: при ошибке API показываются локальные правила
- [ ] Ответы AI релевантны и не содержат "сырых" промптов/ключей

## 🔀 Git & Культура кода
- [ ] Ветка `feat/ai-integration`, атомарные коммиты
- [ ] Обработка исключений `try/except` покрыта логами
- [ ] Кэширование предотвращает повторные дорогие запросы
- [ ] Код AI-логики изолирован в `ai_service.py`
- [ ] PR создан, добавлены примеры промптов в `docs/prompts.md`

## ✅ Чек-лист готовности
- [ ] AI-ответы стабильны, ключи не попадают в Git/логи
- [ ] Приложение не падает при отключении интернета/ошибке API
- [ ] Понятен принцип работы LLM, токены, контекстное окно
- [ ] В `DEVLOG.md` записаны удачные/неудачные промпты и выводы
